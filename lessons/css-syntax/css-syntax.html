What is CSS Syntax?

CSS (Cascading Style Sheets) is a language for describing how HTML elements should look. To write CSS, you need to understand its basic building blocks:

Think of CSS like giving instructions to a browser: "Here's what element to target, and here's how to style it."

CSS Declarations

A declaration is the smallest meaningful unit in CSS. It's a property and a value separated by a colon (:).

Syntax: property: value;

Example declarations:

color: blue;
font-size: 16px;
background: #fff;
margin: 10px;

CSS Declaration Blocks

A declaration block is a group of declarations wrapped in curly braces { }. Declarations inside are separated by semicolons.

Syntax: {
  property1: value1;
  property2: value2;
  property3: value3;
}

Example declaration block:

{
  color: white;
  background: navy;
  padding: 20px;
  border-radius: 5px;
}

It's considered good style to end even the last declaration in a block with a semicolon (e.g., padding: 20px; instead of padding: 20px). This prevents mistakes when adding new declarations later.

CSS Rulesets (Rules)

A ruleset links a selector to a declaration block. It tells the browser: "Apply these declarations to elements matching this selector."

Syntax: selector {
  property: value;
  property: value;
}

Example rulesets:

/* Target a class */
.button {
  background: green;
  color: white;
  padding: 10px 20px;
}

/* Target an element */
p {
  line-height: 1.6;
  margin: 15px 0;
}

/* Target by ID */
#header {
  background: #333;
  color: white;
}

CSS Statements

A statement is a complete CSS instruction that either: a ruleset (selector + declaration block) or an @-rule (meta-information like imports or media queries).

Types of statements:

1. Rulesets (Regular Rules)

The most common type. A selector followed by a declaration block:

/* This is a statement (a ruleset) */
button {
  background: blue;
  border: none;
  cursor: pointer;
}

2. At-Rules (@-Rules)

Special statements that start with @. They convey meta-data or conditional information:

Media Query (responsive design): @media (max-width: 600px) {
  body {
    font-size: 14px;
  }
}
Import (load external styles): @import url('fonts.css');
Font-Face (custom fonts): @font-face {
  font-family: 'MyFont';
  src: url('myfont.woff2') format('woff2');
}
ðŸŽ¯ Remember

Every CSS file is essentially a list of statements. Each statement is either a ruleset or an @-rule. Anything else is ignored by the browser.

Complete Example: Putting It All Together

Here's how declarations, blocks, rulesets, and statements work together:

A complete CSS file: /* Statement 1: Import external styles */
@import url('reset.css');

/* Statement 2: Ruleset for body */
body {
  font-family: Arial, sans-serif;
  margin: 0;
  padding: 0;
}

/* Statement 3: Ruleset for headings */
h1, h2, h3 {
  color: #333;
  font-weight: bold;
}

/* Statement 4: Media query (responsive) */
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
}

Key Takeaways

  • Declarations are the building blocks: property and value separated by a colon.
  • Declaration blocks group multiple declarations together inside { }.
  • Rulesets combine a selector with a declaration block to target elements.
  • Statements are the complete CSS instructions (rulesets or @-rules).
  • Always end declarations with a semicolon (;).
  • Understanding this structure helps you write cleaner, more maintainable CSS!
