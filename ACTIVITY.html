Activity: Semantic Blog Layout with Flexbox, CSS Syntax, and Combinators

Course: Web Systems and Technologies

Total Points: 50

Objective

This activity helps you:

  • Practice semantic HTML structure
  • Apply CSS Flexbox for layout
  • Use correct CSS syntax (selectors + declarations)
  • Apply CSS combinators to target elements cleanly

Focus on structure and layout, not heavy animations.

Activity Description

Create a simple blog-style webpage using:

  • Semantic HTML elements
  • CSS Flexbox for the main layout (required)
  • CSS combinators (required)
  • Correct CSS syntax (required)

You may:

  • Reuse your activity from: https://xavierateneo.instructure.com/courses/2255/assignments/5125
  • Redesign the layout and content to show your own effort

Required Semantic Elements

Your layout must include:

  • header
  • nav
  • main
  • section and/or article
  • aside
  • footer

Layout Requirements (Flexbox Only)

You must use:

  • Flexbox for the main layout (required)
  • Responsive behavior (required) using a media query or flexible units

Your page must include:

  • A header with a site title (blog name)
  • A navigation menu
  • A main content area with blog posts (use article for each post)
  • A sidebar (aside) with supporting content (about, categories, etc.)
  • A footer with basic site info (copyright or contact)

CSS Syntax Refresher (Required)

Write clean CSS using proper selector + declaration block syntax.

Example:

/* Selector targets elements */
article h2 {
  color: #222;
  margin-bottom: 0.5rem;
}

CSS Combinators (Required)

Use at least 4 different combinators in your CSS. You must demonstrate all of these:

  • Descendant selector: A B
  • Child selector: A > B
  • Adjacent sibling selector: A + B
  • General sibling selector: A ~ B

Include short comments in your CSS to label the combinators and syntax you are using so the instructor can quickly see them.

Example usage:

/* Descendant */
main article p {
  line-height: 1.6;
}

/* Child */
nav > ul {
  display: flex;
  gap: 1rem;
  list-style: none;
}

/* Adjacent sibling */
h2 + p {
  color: #444;
}

/* General sibling */
h2 ~ ul {
  padding-left: 1.2rem;
}

Design Expectations

  • Clean spacing and alignment
  • Proper use of margins and padding
  • Readable typography
  • Logical heading hierarchy (h1 then h2, etc.)

You are not graded on artistic design, but layout clarity matters.

Submission Instructions

    • HTML with the required semantic elements
    • CSS using Flexbox for layout
    • CSS combinators (all four types)
    • Responsive behavior (at least one breakpoint)

Grading Rubric (50 Points)

1. Semantic HTML Structure (12 pts)

  • 10-12: Proper and meaningful use of semantic elements
  • 7-9: Mostly correct usage with minor issues
  • 4-6: Some misuse or missing semantic tags
  • 0-3: Poor or incorrect structure

2. Flexbox Layout Implementation (12 pts)

  • 10-12: Layout works correctly and shows clear understanding of Flexbox
  • 7-9: Functional layout with minor alignment or sizing issues
  • 4-6: Layout partially working or inconsistent use of Flexbox
  • 0-3: Minimal or incorrect Flexbox usage

3. CSS Syntax Accuracy (8 pts)

  • 7-8: Clean, correct selector and declaration syntax throughout
  • 5-6: Minor syntax issues that do not break layout
  • 3-4: Multiple syntax errors that affect styling
  • 0-2: Frequent or major syntax errors

4. CSS Combinators Usage (8 pts)

  • 7-8: All four combinators used correctly and meaningfully
  • 5-6: Three combinators used correctly or minor misuse
  • 3-4: Two combinators used or several incorrect uses
  • 0-2: Missing or incorrect combinator use

5. Responsiveness and Cleanliness (10 pts)

  • 9-10: Clear responsive behavior with clean, organized code
  • 6-8: Some responsiveness and mostly organized code
  • 3-5: Minimal responsiveness or cluttered code
  • 0-2: Not responsive and disorganized

Important Notes

  • This is an individual activity.
  • Layout logic is more important than heavy styling.
